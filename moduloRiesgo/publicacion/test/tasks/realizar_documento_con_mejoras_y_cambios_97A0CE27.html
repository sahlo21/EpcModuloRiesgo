Task: Realizar documento con mejoras y cambios
El documento que se genero con anterioridad se establece como un informe general de los cambios que se pueden realizar dentro de la aplicacion.
Purpose
El proposito de esta tarea es realizar un informe detallado con todas las posibles mejoras o cambios de la aplicacion.
Relationships
RolesPrimary Performer:
  • Gerente del proyecto
Additional Performers:
  • Tester QA
InputsMandatory:
  • Informe de proyecto
    		Optional:
    • None
    Outputs
    • Informe de proyecto
      Main Description

      La tarea de realizar el documento con mejoras y cambios se basa en tomar toda el conocimiento obtenido y dejarlo de manera escrita, este documento debe ser entendible y muy detallado, con el fin de evitar que se generen dudas o especulaciones.

      Steps
      Utilizar el bosquejo anterior
      Se actualiza el documento anterior con la informacion nueva que se tiene y de manera detallada.
      Crear el informe del proyecto
      Se detalla cada mejora o cambio que se puede realizar, escribiendo este informe de manera detallada y precisa.
      Notificar a los desarrolladores
      Con el informe realizado se envia este a los encargados de realizar las modificaciones, es decir, los desarrolladores BackEnd y FrontEnd.
