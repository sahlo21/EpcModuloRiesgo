Task: Revisar el plan de proyectos
Se analiza y revisa la arquitectura de software con el fin de obtener la informacion nueva de la arquitectura.
Purpose
El proposito de esta tarea es obtener una arquitectura de software correcta sin errores.
Relationships
RolesPrimary Performer:
  • Arquitecto
Additional Performers:
  • Gerente del proyecto
InputsMandatory:
  • Documento del proyecto
    		Optional:
    • None
    Outputs
    • Documento revision del proyecto
      Main Description

      La tarea de revisar el plan de proyectos se basa en analizar la arquitectura de software con el fin de obtener cambios en la arquitectura y de esta manera evitar errores futuros.

      Steps
      Analizar la informacion de la arquitectura

      Se obtiene la arquitectura de software y en base a esta se analiza la informacion.

      Evidenciar los posibles errores

      Luego de analizar la arquitectura de software se buscan erroes y si se tienen se escriben y luego se notificaran, si no se tiene simplemente se realiza el siguiente paso.

      Notificar al arquitecto
      Se notifica al arquitecto acerca de los posibles errores que se tengan, si no se tiene algun error simplemente se notifica que la arquitectura de software esta en perfecto estado.
