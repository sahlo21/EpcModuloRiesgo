Task: Realizar documentacion
Luego de investigar acerca del proyecto se realizar el compilado de informacion y se desarrolla la arquitectura.
Purpose
El proposito de esta actividad es realizar el proceso de la arquitectura de software.
Relationships
RolesPrimary Performer:
  • Arquitecto
Additional Performers:
InputsMandatory:
  • Documento del proyecto
    		Optional:
    • None
    Outputs
    • Documento del proyecto
      Main Description

      La tarea de realizar la documentacion se basa en la creacion de la arquitectura del documento de software, es decir, se realiza la arquitectura de software, y por ende se realizan los diagramas y documentos necesarios.

      Steps
      Analizar la informacion
      Luego de obtener toda la informacion se analiza esta y se obtiene la necesaria para empezar a realizar el proceso de arquitectura de software.
      Desarrollar la arquitectura

      En este paso se empieza a desarrollar la arquitectura del software, es decir, se empieza a realizar las reglas de negocio, diagramas de caso de uso, requisitos y los diversos diagramas como lo son el espina de pescado y diagrama de parapeto.
