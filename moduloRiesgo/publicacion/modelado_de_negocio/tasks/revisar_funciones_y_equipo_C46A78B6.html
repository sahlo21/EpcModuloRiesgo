Task: Revisar funciones y equipo
Esta tarea se encarga de realizar una revision sobre las funciones y el equipo de trabajo con el fin de evitar errores en futuros procesos de la arquitectura de software.
Purpose
El proposito de esta tarea es evitar errores con respecto a actividades realizadas con anterioridad.
Relationships
RolesPrimary Performer:
  • Gerente del proyecto
Additional Performers:
  • Arquitecto
InputsMandatory:
  • Brief del proyecto
    		Optional:
    • None
    Outputs
    • Acta de inicio
      Main Description

      La tarea de revisar funciones y equipo se realiza con el fin de evitar arrastrar errores a otras actividades, es decir, si se tiene un error en la actividad anterior se notifica con el fin de realizar una arquitectura de software errada.

      Steps
      Analizar la informacion de la actividad anterior
      En este paso se analiza toda la informacion que se obtuvo en la actividad anterior, con el fin de verificar que esta sea correcta o evidenciar los errores que se tengan.
      Gestionar si se tiene errores
      Si la actividad anterior tiene errores se notifica y modifica con el fin de tener informacion valida, si por el contrario no tiene errores se procede a realizar la siguiente actividad.
